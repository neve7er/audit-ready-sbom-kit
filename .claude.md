## Source of Truth
`docs/architecture.md` is the single source of truth for this project. When any instruction conflicts with `docs/architecture.md`, stop and flag the conflict before proceeding.

## Core Coding Contract
All code in `src/` must follow these rules without exception:

1. **Types first** — every value must conform to a type defined in `src/core/sbom/cyclonedx/model.ts`
2. **Pure functions only in `src/core/`** — no side effects, no I/O, no global state mutation
3. **Side effects are adapter-only** — file reads, HTTP calls, and cache writes belong exclusively in `src/adapters/`

## Deterministic Constraint (Strategic Pivot)
These rules apply to all Phase 1 implementation and override any prior instruction:

- **No LLM Logic** — no probabilistic scoring, no AI-generated rationale, no non-deterministic classification of any kind. Every verdict must be reproducible from identical inputs.
- **PURL Sovereignty** — PURL generation uses `src/core/utils/purl.ts → buildPurl()` exclusively. `encodeURIComponent`, `URL`, and all standard library encoders are forbidden for this function.
- **Immutable Contract** — every data type inside `src/core/` carries `readonly` on all fields and arrays. No post-construction mutation is permitted.

## Inviolable Rules
- Never write logic in `src/core/` that imports from `src/adapters/`
- Never write a function in `src/core/` that cannot be unit-tested without mocking
- Never introduce a dependency without explicit human approval

## Stop Conditions
Stop and ask the human before:
- Modifying `src/core/sbom/cyclonedx/model.ts` (schema changes affect the entire pipeline)
- Adding any new `npm` dependency
- Creating any file outside the scaffolded structure
- Resolving any ambiguity by assumption — ask instead
- Producing any encoding for `@` or `/` in a PURL that does not go through `src/core/utils/purl.ts → buildPurl()`
- Writing any code inside `src/core/` that mutates a field or array after construction (violates `readonly`)
- Adding any classification or verdict logic where `reasonCode` is undefined or omitted
- Running any scan when `.audit-policy.json` fails schema validation or contains an expired exception entry
- Introducing any time-dependent or environment-dependent input into `--fail-on` classification logic (current time, env vars, etc.)
- Silently ignoring a user-supplied exception where `reason` is fewer than 20 characters or is a placeholder — prompt for a specific justification instead
